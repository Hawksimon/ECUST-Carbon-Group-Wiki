# 5 编写与维护Wiki

本课题组 Wiki 采用 **MkDocs** 框架及 **Material for MkDocs** 主题构建。建立本 Wiki 的初衷是系统化沉淀课题组内的实验技术、规章制度与基础知识，实现知识的高效传承。

为了方便组内所有同学（尤其是无编程背景的同学）参与知识库的共建，本系统已在后台配置了基于 GitHub Actions 的全自动部署工作流。你无需在本地安装任何编译环境，仅需通过浏览器即可完成内容的撰写、修改与发布。

---

## 5.1 在线编辑与提交流程

本 Wiki 的所有源代码托管于 GitHub 平台。日常内容的更新推荐采用网页端直接修改的方式。

**1. 获取协作权限**
首次参与编写前，请注册个人的 GitHub 账号，并将用户名告知 Wiki 现任管理员，申请加入该仓库的协作者（Collaborators）列表。

**2. 网页端修改文件**
1. 访问课题组的 GitHub 仓库页面，进入 `docs` 文件夹，找到你需要修改的 Markdown (`.md`) 文件。
2. 点击页面右上角的**铅笔图标（Edit this file）**，进入文本编辑模式。
3. 在文本框内使用 Markdown 语法进行内容的增删改查。

**3. 提交修改 (Commit)**
编辑完成后，点击页面右上方的绿色按钮 **Commit changes...**。
在弹出的对话框中，请简明扼要地填写本次修改的内容摘要（例如：`Update: 更新管式炉操作规范`），然后点击 **Commit changes** 确认提交。

> **部署说明：** 提交成功后，云端自动化脚本将立即启动。通常在 1~2 分钟内，最新的网页即可自动编译并发布，你只需刷新 Wiki 网站即可查看更新。

## 5.2 基础 Markdown 语法指南

Markdown 是一种轻量级标记语言，通过简单的纯文本符号即可实现结构化的排版。编写本 Wiki 时，请遵循以下基础语法：

### 标题
使用 `#` 号标识标题层级，`#` 后**必须保留一个空格**。合理使用标题有助于系统自动生成页面右侧的目录导航。

```markdown
# 一级标题（通常用于页面主标题，单页仅限一个）
## 二级标题
### 三级标题
```

### 文本强调

```markdown
使用 **双星号** 可以对文字进行加粗。
使用 *单星号* 可以对文字进行斜体强调。
```

### 列表
* **无序列表**：使用 `-` 或 `*` 加上空格。
* **有序列表**：使用数字加英文句点，例如 `1. `。

### 超链接与图片

```markdown
[点击访问华理财务处官网](https://cwc.ecust.edu.cn)
![设备图片说明](images/equipment.jpg)
```
*(注：建议将页面所需图片统一放置于 `docs/images` 文件夹下，并使用相对路径引用。)*

## 5.3 高阶排版特性 (Material 主题扩展)

得益于 Material 主题的强大扩展，本 Wiki 支持多种高级排版元素，有助于提升文档的易读性与专业度。

### 1. 提示块 (Admonitions)
在撰写实验安全或操作警示时，强烈建议使用提示块。格式为 `!!!` + 类型 + "标题"。
**注意：提示块内部的文本必须进行 4 个空格的缩进。**

```markdown
!!! note "实验记录规范"
    请务必在实验当天完成实验记录本的填写。

!!! warning "温度限制"
    水热釜的加热温度严禁超过 200 °C，否则有爆炸风险。

!!! danger "危险化学品操作"
    使用氢氟酸必须在通风橱内进行，并佩戴专用防护装备。
```

### 2. 任务列表
常用于实验步骤或入组流程的核对清单。

```markdown
- [x] 完成实验室安全培训
- [ ] 拥有自己的实验室柜子
```

### 3. 代码块
在记录命令行操作或参数配置时，请使用反引号将代码包裹，并可标明语言以实现语法高亮。

```bash
# 提交计算任务的命令
sbatch job.sh
```

## 5.4 新增页面与导航栏配置

若你撰写了一篇全新的文档，仅在 `docs` 文件夹下创建 `.md` 文件是不够的，必须将其注册到系统的全局导航树中。

1. **新建文件**：在 GitHub 网页端的 `docs` 目录下，点击 `Add file -> Create new file`。**文件名强烈建议使用纯英文或拼音**（如 `sem-guide.md`），以避免生成乱码链接。
2. **修改配置文件**：返回仓库根目录，打开 `mkdocs.yml` 文件。
3. **注册导航**：向下滚动定位至 `nav:` 节点，严格按照现有的缩进层级，添加你的新文件路径。

!!! danger "YAML 语法缩进警告"
    `mkdocs.yml` 文件对格式要求极其严格。
    在添加新导航条目时，**严禁使用 Tab 键**，必须使用空格键进行层级对齐。且冒号 `:` 之后必须保留一个空格。

**配置示例：**

```yaml
nav:
  - 索引首页: index.md
  - 实验安全: 0_safety.md
  - 仪器操作说明:
      - 离心机使用指南: sem-guide.md  # 此处为你新增的页面
```
修改完成后，提交 `mkdocs.yml` 文件，系统编译后你的新页面即会出现在网站的左侧导航栏中。

## 5.5 维护规范原则

为保证 Wiki 知识库的长期质量与整洁，请各位贡献者共同遵守以下原则：

1. **排版规范**：注意中英文之间的空格，统一使用全角中文标点符号。

2. **客观严谨**：实验参数、安全规范等核心数据必须经过核实再行录入。

3. **尊重协同**：如需对他人编写的核心板块进行大幅修改，请事先在课题组内沟通确认。
