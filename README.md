# ECUST Carbon Group's Wiki

欢迎来到华东理工大学碳材料课题组的官方 Wiki 知识库项目源码仓库！

本 Wiki 采用 [MkDocs](https://www.mkdocs.org/) 框架及 [Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 主题构建，旨在系统化沉淀课题组内的实验技术、规章制度与基础知识，实现知识的高效传承。

## 📂 项目文件夹与文件说明

本项目在本地或云端的结构及其功能如下：

* **`docs/`**
  网站的核心内容文件夹。所有网页的纯文本底稿（`.md` Markdown 文件）以及用到的图片资料（`images/`）都存放在这里。**日常更新 Wiki 只需要修改此文件夹内的内容。**
* **`overrides/`**
  自定义覆盖文件夹。用于存放修改官方主题默认样式的代码，例如我们在这里通过 `partials/comments.html` 植入了 Giscus 评论区功能。
* **`.github/`**
  GitHub 自动化机器人配置文件。其中的 `workflows/deploy.yml` 脚本赋予了本仓库“全自动发布”的能力。只要有代码推送到主分支，机器人就会自动构建并发布最新网页。
* **`site/`**
  本地编译生成的网页输出文件夹。当你在本地运行预览时，MkDocs 会自动生成此文件夹。它不需要上传到 GitHub（通常会被 Git 忽略）。
* **`mkdocs.yml`**
  **网站的总配置文件（最高机密）。** 包含网站名称、颜色主题、页面左侧导航栏（`nav`）的层级结构、以及各种增强插件的开启开关。**若在 `docs/` 下新建了文章，必须在此文件的 `nav` 列表中报备，否则网页导航栏不会显示。**
* **`README.md`**
  即本文件，项目的说明书。

## 🚀 如何从本地上传与更新 Wiki？

感谢自动化部署（GitHub Actions）的加持，更新网站变得非常简单。请遵循以下标准工作流：

### 第一步：在本地修改内容
在 `docs/` 文件夹中修改现有的 `.md` 文件，或新建 `.md` 文件（若新建，请记得同步修改 `mkdocs.yml` 中的 `nav` 列表）。

你可以随时在终端运行以下命令进行本地实时预览，确认无误后再进行上传：

```bash
mkdocs serve
```
*(运行后在浏览器访问 `http://127.0.0.1:8000` 即可预览)*

### 第二步：提交并推送到 GitHub（备份源码并触发自动发布）
在确认修改无误后，打开终端（确保路径处于项目根目录），依次执行以下三条命令：

```bash
# 1. 追踪所有修改过的文件
git add .

# 2. 提交修改，并写下简短的更新说明（请替换引号内的文字）
git commit -m "Update: 新增了 SEM 仪器使用指南"

# 3. 将源码推送到云端仓库
git push
```

### 第三步：喝杯茶，等待上线
执行完 `git push` 后，你的工作就彻底结束了。
云端自动化机器人会在后台自动接管任务。等待大约 1~2 分钟，直接刷新我们的 Wiki 官方网站，即可看到最新的内容！
